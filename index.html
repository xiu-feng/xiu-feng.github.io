如何实现分表

MySQL数据库每个表的数据最好是小于1000万，大于1000万会有明显的查询效率问题。那么我现在有一个业务，比如订单服务，并发量很大。我该怎么来实现存储？

创建 tb_order_0,tb_order_1,tb_order_2,tb_order_3,tb_order_4,tb_order_5,tb_order_6,tb_order_7,tb_order_8,tb_order_9.十个表，数据id使用long类型。id规则为当前的时间戳加上随机的三位数，比如2020041911185834031.那么这条数据存储就按照最后一位数字来存储到对应的表中，前面数字的最后一位为1，则将这个数据存放在tb_order_1中。查询也是的根据id去不同的表中查询。

当然如果数据量还要更大的还可以继续分表。如 tb_order_0_0 十个表都不够了，你就发财了，嘻嘻。

例： select * from tb_order_${num} where id=#{id},num为id的最后一位

数据必须归档，制定一个归档策略。
